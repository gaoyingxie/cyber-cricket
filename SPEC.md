## 一、设计理念

### 1.1 核心概念

**「赛博斗龙虾」** 是一款以虾苗养成为核心、**全自动策略战斗**为骨架的roguelike循环游戏。玩家在「养虾模式」中从一只随机技能组合的幼虫开始，通过5轮战斗逐步养成强大的究极龙虾；在「成虾导入」中可以继承已有龙虾继续挑战高难度敌人。

**核心特点：完全自动战斗**，玩家只需观看龙虾自动战斗、积累技能、搭配Build，享受「赛博斗蛐蛐」的乐趣。

游戏的核心乐趣在于：
- **Build构筑**：不同技能组合带来截然不同的战斗体验
- **资源博弈**：HP管理、技能冷却、进化时机
- **信息博弈**：敌人技能随机，战斗充满未知
- **数值成长**：从弱小的幼虫进化为究极龙虾的成就感

### 1.2 设计关键词

- **快速循环**：每局游戏5轮，15-20分钟可完成
- **永久养成**：成虾代码可导出保存，下局可继续使用
- **策略深度**：18个主动技能 + 9个被动技能，组合丰富
- **视觉反馈**：攻击动画、伤害数字、暴击特效、进化特效

### 1.3 目标用户

- 喜欢养成类游戏的玩家
- 喜欢策略战斗（技能搭配）的玩家
- 喜欢短平快roguelike循环的玩家

```

│       ↓           ↓           ↓           ↓                 │
│    模式选择    随机生成属性   展示龙虾    敌人选择            │
```

## 四、战斗逻辑详解

### 4.1 回合行动规则

**玩家回合 (executeAutoTurn)**：
1. 检查眩晕/封印状态
2. 70%概率释放随机可用技能，30%概率直接攻击
3. 技能有冷却则不能释放
4. 被动技能自动生效（暴击、连击、吸血等）

**敌人回合 (enemyTurn)**：
1. 检查眩晕/封印状态
2. 释放所有可用技能（按技能数组顺序）
3. 最后执行普通攻击
4. 被动技能自动生效

**回合结束 (endTurn)**：
1. 所有技能冷却-1
2. DOT伤害结算（中毒、流血）
3. 30%概率解除降防状态
4. 眩晕/封印状态清除
5. 刷新被动技能加成

### 4.2 伤害计算

```
最终伤害 = max(1, floor(攻击方攻击力 × 技能倍率 × 暴击倍数 × (1 - 防御减伤)))
防御减伤 = def / (dmg + 100)
```

**护盾吸收**：伤害先减护盾层数，再计算防御
**减伤效果**：叠加降低最终伤害
**无视防御**：部分技能（如护盾击破）直接计算伤害

### 4.3 速度与先手

当前版本**没有严格的速度先手逻辑**，速度属性仅作为显示。未来可扩展为：
- 速度高者先行动
- 速度影响DOT伤害频率

### 4.4 胜负判定

| 条件 | 结果 |
|------|------|
| 敌人HP ≤ 0 | 玩家胜利，进入奖励结算 |
| 玩家HP ≤ 0 | 玩家失败，游戏结束 |
| 神佑被动触发 | 复活并恢复50%HP，继续战斗 |

### 6.2 敌人属性计算

```
基础难度系数 = 0.4 + (round-1) × 0.15
每轮递增系数 = 1 + (round-1) × 0.15
实际难度系数 = 每轮递增系数 × 基础难度系数

HP = floor((100 + 轮数×15) × 难度系数)
ATK = floor((7 + 轮数×1.3) × 难度系数)
DEF = floor((3 + 轮数×0.8) × 难度系数)
SPD = floor((5 + 轮数×0.6) × 难度系数)
```

### 6.3 敌人技能数量

| 轮数 | 敌人1/2技能数 | 敌人3技能数 |
|------|-------------|------------|
| 第1轮 | 1 | 2 |
| 第2轮 | 2 | 3 |
| 第3轮 | 3 | 4 |
| 第4轮 | 4 | 5 |
| 第5轮 | 5 | 6 |

### 6.4 难度曲线

```
round1: 40% → round5: 100%
敌人属性随轮数指数增长
玩家需要通过装备+技能偷学+进化来应对
```

## 七、数值体系

### 7.1 玩家初始属性

```
HP = floor(110 + random(0,20))
ATK = floor(9 + random(0,3))
DEF = floor(4 + random(0,3))
SPD = floor(5 + random(0,8))
技能数 = 2 (90%概率) 或 3 (10%概率)
```

### 7.2 等级成长

```
每升级:
HP增加: +15 × 阶段系数
ATK增加: +2 × 阶段系数
DEF增加: +1 × 阶段系数
SPD增加: +1 × 阶段系数
```

### 7.3 阶段（进化）

| 阶段 | 名称 | 等级要求 | 属性系数 | 精灵 |
|------|------|---------|---------|------|
| 0 | 虾苗 | Lv.1-2 | 1.0x | 🦐 |
| 1 | 幼虾 | Lv.3-4 | 1.15x | 🦞 |
| 2 | 战斗虾 | Lv.5+ | 1.3x | ⚡🦞 |
| 3 | 铁甲虾 | - | 1.6x | 🔱🦞 |
| 4 | 究极虾 | - | 2.0x | 👑 |

### 7.4 装备数值

```
装备价值 = 基础值 × 品质系数 × 等级
ATK装备基础值: 3
DEF装备基础值: 2
HP装备基础值: 20

品质系数: 普通=1.0, 精良=1.5, 稀有=2.0, 史诗=3.0, 传说=5.0
装备等级: 1 + min(floor(轮数/2), 10)

ATK装备增加: 装备价值
DEF装备增加: 装备价值
HP装备增加: 装备价值 × 5
```

### 7.5 掉落概率

```
装备掉落率: 40%
品质分布: random(0, min(3 + round×0.3, 5))
```

## 九、养成系统

### 9.1 进化规则

| 条件 | 进化 | 奖励 |
|------|------|------|
| Lv.2 → Lv.3（第2轮结束时） | 虾苗→幼虾 | 属性×1.15 + 随机技能1个 |

### 9.2 技能偷学

- 每场战斗胜利：100%偷学敌人1个随机技能
- 已有技能不重复获取
- 进化时额外获得1个随机新技能

### 9.3 成虾导出

- 通关后（5轮胜利）可导出成虾代码
- 代码格式：Base64编码的JSON
- 包含：等级、阶段、属性、技能、装备、背包
